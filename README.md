# Cinema Web — Онлайн-кинотеатр для TorrServer

**Cinema Web** — это современный, быстрый и адаптивный веб-интерфейс в стиле Netflix (Netflix Clone), созданный для поиска, каталогизации и стриминга фильмов и сериалов непосредственно с вашего домашнего сервера **TorrServer**.

Проект работает локально, не требует регистрации и ориентирован на развертывание на домашнем NAS (например, под управлением Traefik).

---

## 🚀 Основные возможности

* **Стриминг торрентов «на лету»:** Онлайн-просмотр во встроенном HTML5-плеере или запуск в один клик во внешнем плеере (VLC, Infuse и др.) без необходимости предварительного скачивания.
* **Собственная медиатека TorrServer:**
  * Автоматический вывод всех добавленных в TorrServer торрентов на главной странице.
  * Раздел **«Недавно запущенные»** для быстрого возобновления просмотра.
  * Прямой запуск: если торрент уже добавлен в TorrServer, плеер откроется мгновенно со списком серий, минуя поиск раздач.
* **Параллельный двуязычный поиск (Jackett):** При поиске зарубежного контента система выполняет параллельные запросы на русском и английском языках, дедуплицируя и сортируя результаты по сидам.
* **Умная фильтрация мусора:** Автоматическое отсечение раздач без видео-тегов качества (1080p, 4K и т.д.), исключение курсов, книг, журналов, музыки, архивов (`.zip`, `.rar`) и фейковых раздач.
* **Автопропуск заставок (IntroDB & AniSkip):** Интеграция с API таймингов для автоматического пропуска вступительных заставок (интро) и переключения на следующую серию при начале титров (аутро).
* **Группировка файлов по сезонам:** Полная визуальная группировка файлов торрента по сезонам (табы выбора сезона) с автоматическим переключением активной вкладки во время автопроигрывания.
* **Локальное кэширование:**
  * Кэширование `.torrent` файлов на диске (Next.js) для исключения повторных запросов к трекерам и защиты от блокировок по IP.
  * Кэширование результатов поиска Jackett с TTL 3 часа.

## 🛠️ Архитектура

```text
[ Браузер (Клиент) ] ➔ [ Cinema Web (Next.js) ]
                             │
                             ├─➔ [ TMDB API ] (Каталог фильмов и метаданные)
                             ├─➔ [ Jackett API ] (Поиск торрент-раздач на трекерах)
                             ├─➔ [ TorrServer API ] (Управление торрентами и стриминг)
                             └─➔ [ AniSkip / IntroDB API ] (Тайминги автопропуска)
```

## 🗺️ Технический Roadmap (План развития)

Ниже представлен детальный план развития проекта, сформированный по результатам архитектурного анализа и согласованный с концепцией stateless-приложения.

### **1. Локальное хранение и прогресс воспроизведения (Stateless / Client-Side)**
- [x] Хранение локальной истории открытых фильмов.
- [ ] **IndexedDB хранилище (Local History & Progress):** Интеграция базы данных IndexedDB на стороне клиента (через `localForage` или чистый API) для сохранения:
  * Прогресса воспроизведения (таймкоды с точностью до секунды) для каждой серии/фильма.
  * Истории просмотров с датами запусков.
  * Списков «Избранное» и «Смотреть позже».
  * Индивидуальных настроек плеера (громкость, выбранный язык дорожки и субтитров).
- [ ] **Синхронизация прогресса:** Сохранение прогресса изолированно для каждого устройства/браузера (без серверной СУБД для обеспечения полной конфиденциальности данных).

### **2. Оптимизация стриминга и буферизации (Smart Prebuffering)**
- [x] Автоматическое переключение на следующую серию по окончании текущей.
- [x] Интерактивный оверлей автопропуска интро/аутро с таймером отмены.
- [x] Группировка файлов торрента по сезонам (как в TorrServer Matrix).
- [ ] **Фоновая предзагрузка (Prebuffering):** 
  * Отслеживание времени окончания серии в видеоплеере.
  * Автоматический фоновый запуск предзагрузки следующего эпизода в TorrServer (через API `/torrent/action` с `action: "preload"`), когда до конца текущей серии остается менее 5 минут или при появлении оверлея пропуска титров. Это обеспечит бесшовный запуск следующего файла без ожидания буферизации.

### **3. Совместимость форматов и кодеков (Media Analysis)**
- [ ] **Анализ кодеков файлов перед воспроизведением:**
  * Интеграция проверки поддержки видео- и аудиокодеков на клиенте (через `MediaSource.isTypeSupported` и анализ структуры потоков торрента).
  * Если файл содержит неподдерживаемый браузером кодек (например, HEVC/H.265 видео, DTS или E-AC3/Dolby Digital Plus аудио на ПК/iOS):
    * Выводить понятное предупреждение пользователю в окне плеера.
    * Отображать кнопку быстрого запуска во внешнем плеере (VLC/Infuse/Kodi), который декодирует эти форматы аппаратно на стороне клиента.
- [x] Генерация ссылок для быстрого открытия потоков во внешнем VLC-плеере (`vlc://`).

### **4. Работа с аудиодорожками и субтитрами (Tracks Management)**
- [ ] **Интеграция внешних субтитров из торрента:**
  * Парсинг структуры торрент-файла и автоматический поиск файлов субтитров (`.srt`, `.vtt`, `.ass`) в директории проигрываемой серии.
  * Автоматическое подключение найденных файлов к HTML5-плееру через тег `<track>`.
- [ ] **Кастомный пульт управления дорожками в плеере:**
  * Вывод меню выбора встроенных аудиодорожек и встроенных субтитров непосредственно в интерфейс видеоплеера (при наличии поддержки стриминга отдельных аудиодорожек браузером).

### **5. Модульная система метаданных (Multi-Catalog Metadata)**
- [x] Интеграция каталога TMDB для фильмов и сериалов.
- [ ] **Модульный бэкенд метаданных:**
  * **TMDB:** Дефолтный источник для зарубежных фильмов и сериалов.
  * **Shikimori API:** Автоматическое переключение каталога при поиске аниме для вывода корректных русскоязычных описаний, оригинальных постеров и точного деления на сезоны, спешлы и OVA.
  * **Кинопоиск (Неофициальный API):** Использование в качестве источника для российских фильмов и сериалов для обеспечения наиболее полных описаний и актуальных рейтингов.

### **6. Панель администрирования TorrServer (TorrServer Dashboard)**
- [ ] **Административный дашборд:** Создание отдельной вкладки управления TorrServer внутри интерфейса Cinema Web:
  * Отображение статуса дискового пространства и размера кэша TorrServer.
  * Управление списком торрентов (удаление раздач, очистка кэша, запуск и остановка торрентов для освобождения места на сервере).
  * Редактирование сетевых настроек TorrServer (лимиты кэша, буфера предзагрузки, максимальное число соединений и пиров) непосредственно через API.

## 📦 Запуск и развертывание

### 1. Настройка окружения

Создайте в корневой директории проекта файл `.env.local`:

```ini
# URL этого веб-приложения для внешнего доступа
NEXT_PUBLIC_APP_URL=https://watch.nas-soft.com

# API Ключ TMDB (The Movie Database)
TMDB_API_KEY=YOUR_TMDB_API_KEY

# Адрес API Jackett (внутренний адрес в сети Docker) и его API Key
JACKETT_API_URL=http://jackett:9117
JACKETT_API_KEY=YOUR_JACKETT_API_KEY
JACKETT_ADMIN_PASSWORD=YOUR_JACKETT_ADMIN_PASSWORD

# API TorrServer (внутренний адрес в сети Docker)
TORRSERVER_API_URL=http://torrserver:8090
```

### 2. Развертывание через Docker Compose

Ниже приведен пример конфигурации для интеграции в общую сеть `tweb` с обратным прокси Traefik:

```yaml
services:
  cinema-web:
    build:
      context: ./cinema-web
      dockerfile: Dockerfile
    container_name: cinema-web
    mem_limit: 1g
    restart: unless-stopped
    volumes:
      - ../cinema-web/cache:/app/cache # Персистентный кэш
    env_file:
      - ./cinema-web/.env.local
    networks:
      - tweb
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.cinema.rule=Host(`watch.nas-soft.com`)"
      - "traefik.http.routers.cinema.entrypoints=websecure"
      - "traefik.http.routers.cinema.tls=true"
      - "traefik.http.routers.cinema.tls.certresolver=myresolver"
      - "traefik.http.services.cinema-svc.loadbalancer.server.port=3000"

networks:
  tweb:
    external: true
```

Для сборки и запуска выполните:
```bash
docker compose up -d --build
```
После успешного запуска приложение станет доступно по внешнему адресу с автоматическим SSL-сертификатом от Let's Encrypt.
